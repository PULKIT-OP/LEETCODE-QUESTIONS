# 🚀 DSA Solutions Repository

A curated collection of **Data Structures & Algorithms** solutions from **LeetCode** and **GeeksforGeeks (GFG)**, organized topic-wise for efficient learning, revision, and interview preparation.

This repository is designed not only to provide working solutions but also to help understand the **thought process**, **approach selection**, and **implementation details** behind each problem.

---

## 📚 Repository Features

- 📂 **Topic-wise Organization** for easier navigation and revision.
- 🧠 **Multiple Approaches** for many problems, ranging from brute-force to optimal solutions.
- 📝 **Detailed Comments** explaining the intuition and implementation at every important step.
- 🔗 **Direct Problem Links** included inside each solution file.
- 💡 **Interview-Oriented Solutions** focused on clarity, readability, and learning.

---

## 📂 File Naming Convention

### LeetCode Problems

Format:

```text
LC_<Question_Number>_<Question_Title>.cpp
````

Example:

```text
LC_123_Best_Time_to_Buy_and_Sell_Stock_III.cpp
```

Explanation:

* `LC` → Problem belongs to LeetCode.
* `123` → LeetCode Question Number.
* Remaining part → Question Title.

---

### GeeksforGeeks Problems

Format:

```text
GFG_<Question_Title>.cpp
```

Example:

```text
GFG_Boundary_Traversal_of_Binary_Tree.cpp
```

Explanation:

* `GFG` → Problem belongs to GeeksforGeeks.
* Remaining part → Question Title.

## 🧩 Solution Structure

Each solution file generally follows a structured format.

### Problem Link

Every file contains a direct link to the original problem.

```cpp
// Problem Link: <Direct Question Link>
```

### Multiple Approaches

Many problems contain multiple approaches separated using comment blocks:

```cpp
// METHOD 1: Brute Force Approach

...

// METHOD 2: Better Approach

...

// METHOD 3: Optimal Approach
```

This allows readers to understand how a solution evolves from a simple idea to a more optimized one.

### Detailed Explanation

Most solutions contain detailed comments explaining:

* Why a particular step is performed.
* The purpose of variables and data structures.
* Important observations and edge cases.
* Time and Space Complexity considerations.

Example:

```cpp
// Store the current node's value before moving forward

// This condition handles the edge case when the list is empty

// Update answer because we found a better candidate
```
